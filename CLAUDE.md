# CLAUDE.md — контекст проєкту «У гори»

Цей файл — короткий бриф для Claude Code. Прочитай його на початку сесії.

## Що це за проєкт

Інтерактивний застосунок-помічник для **майстер-класу з підлітками про походи в гори**. Допомагає зібрати правильне спорядження, зрозуміти, який має бути одяг/стаф, переглянути відео-поради, обрати маршрут і запам'ятати головне про безпеку.

**Цільова аудиторія:** підлітки. Тон — простий, дружній, без зайвого тексту.

## Ключові принципи (не порушувати без потреби)

- **Статика, без бекенду й без БД.** Жодних серверних залежностей. Дані користувача (стан чек-листа) — лише в `localStorage`.
- **Mobile-first.** Основний пристрій — телефон. Великі тапи (≥48px), читабельність на сонці (світлий фон, високий контраст).
- **Офлайн (PWA).** У горах часто немає зв'язку. Service worker кешує все, крім зовнішніх ресурсів (відео YouTube).
- **Легке редагування контенту без коду.** Увесь контент винесено в `data.js` — редагується зміною тексту між лапками.
- **Хостинг — GitHub Pages.** Має працювати і при відкритті `index.html` подвійним кліком (`file://`), і на Pages.

## Технічні рішення (і чому саме так)

- **Без збірки, але вже не монофайл.** CSS — у теці `css/` (кілька `.css` через `<link>` по порядку), JS — у теці `js/` (класичні скрипти, що ділять спільний глобальний скоуп; порядок підключення важливий, `main.js` останнім). Це не ES-модулі, тож відкриття подвійним кліком (`file://`) працює. Простий деплой без тулчейну.
- **Контент і мови у `data.js`, а НЕ `data.json`** — навмисно. `fetch` не працює при відкритті подвійним кліком (`file://`, CORS). `data.js` через `<script src>` працює і локально, і на сервері. Двомовно: `window.MOUNTAIN_DATA = { uk:{...}, en:{...} }`, кожна мова має `ui` + контент + `safety`.
- **Ілюстрації маршрутів — власні SVG/фото** (`images/routes/`), щоб уникнути проблем з авторськими правами й працювати офлайн. Іконки — `images/icons/`.
- **Стан чек-листа в `localStorage`** за ПОЗИЦІЄЮ (`день|індекс категорії|індекс пункту`), тож галочки зберігаються при зміні мови. Ключі: `gory-pack-v2`, `gory-day-v1`, `gory-season-v1`, `gory-lang-v1`.

## Структура файлів

```
mountainsApp/
├── index.html      # лише розмітка; підключає css/, data.js, js/
├── data.js         # УВЕСЬ контент + рядки UI, двомовно (uk/en)
├── css/            # base.css, components.css, safety-nav.css
├── js/             # state, gear, controls, render, i18n, main (порядок!)
├── sw.js           # service worker (офлайн-кеш): масив ASSETS + версія CACHE
├── manifest.json   # PWA: назва, іконки, кольори
└── images/
    ├── icons/      # favicon + іконки PWA
    └── routes/     # ілюстрації/фото маршрутів (parashka.svg, hora_lopata.jpg)
```

> Після додавання нового файлу в `css/` чи `js/` — додай його і в `<link>`/`<script>` в `index.html`, і в масив `ASSETS` у `sw.js`.

## Модель контенту (`data.js` → `window.MOUNTAIN_DATA`)

- **`gear`** — об'єкт із ключами `"1"` і `"2"` (дні). Кожен — масив категорій `{cat, items, ...}`.
  - Пункт: рядок `"Ложка"` або об'єкт `{n:"Баф", opt:true}` (`opt` → бейдж «опц.»).
  - Прапорці категорії: `must:true` (помаранчевий блок «Must have»), `season:true` (зимове спорядження, видиме лише в режимі «Холодна пора»), `note:"..."` (підпис під заголовком).
- **`videos`** — масив `{id, title, desc}` (id — частина YouTube-URL після `watch?v=`).
- **`routes`** — масив `{name, area, height, duration, difficulty, days, image, desc, tip}`. `days` → бейдж «1 день/2 дні». `difficulty` визначає колір пігулки (легкий/середній/складний).
- **`routeApps`** — масив `{name, url, desc}` (сервіси пошуку маршрутів).
- **`gearExamples`** — масив `{item, wrong, right, layer?}` (правильно/неправильно; `layer` — для трьох шарів одягу).
- **Безпека** — це статичний HTML у `index.html`, секція `id="safety"` (не в data.js).

## Що вже реалізовано

П'ять розділів (нижня навігація):
1. **Рюкзак** — інтерактивний чек-лист на 1 і 2 дні. Перемикач днів, перемикач «Тепла/Холодна пора» (показує зимове спорядження), прогрес-бар «зібрано X/Y», лічильники по категоріях, кнопка «Скинути». Стан зберігається.
2. **Стаф** — «Як виглядає спорядження»: правило трьох шарів + 8 карток «правильно/неправильно».
3. **Відео** — 2 відео з YouTube (тришарова концепція, регулювання рюкзака) + підказка дивитись завчасно.
4. **Маршрути** — 4 реальні маршрути з фото-ілюстраціями (Парашка, Говерла, Піп Іван, Синевир) + 3 сервіси для пошуку власних.
5. **Безпека** — екстрені номери (112, 101, клікабельні), дії при втраті групи, перша допомога, правила групи.

Плюс: PWA (manifest + service worker + іконки), адаптивність, фокус для клавіатури, підтримка `prefers-reduced-motion`.

## Дизайн-система

- **Палітра:** `--spruce:#163A2E` (бренд/шапка), `--moss:#2E6B4F` (акції/прогрес), `--signal:#F26A1B` (сигнальний помаранчевий — лише для важливого: «Must have», екстрені номери), `--stone:#EEF1ED` (фон), `--paper:#FFFFFF` (картки), `--sky:#3C7DB0` (зимове/сезонне).
- **Шрифт:** системний sans (офлайн-стійкий), особистість — через вагу, розмір, трекінг.
- **Фішка:** топографічні лінії + силует гір у шапці; категорії чек-листа марковані «блейзом» (кольорова смужка, як трекінговий маркер). Зелений = звичайне, помаранчевий = must-have, синій = сезонне.
- Принцип: сміливість в одному місці, решта — спокійна. Не додавати зайвого декору.

## Бэклог — ідеї, що залишилось зробити

Пріоритет приблизний (зверху — найкорисніше).

1. **Реальні фото маршрутів** — замінити SVG у `images/` на справжні знімки (покласти файл і оновити поле `image` в `data.js`). Звернути увагу на авторські права / ліцензії.
2. **Більше маршрутів** — додати ще варіанти (напр. Хом'як, Пікуй, Драгобрат), бажано позначати, для якого варіанту походу (1/2 дні) підходить.
3. **Розділ «Їжа та вода»** — скільки води брати, що їсти на 1 vs 2 дні, перекуси на маршруті.
4. **Фото-приклади в «Стаф»** — додати реальні зображення «правильно/неправильно» до карток (зараз лише текст).
5. **Розділ «Погода»** — як перевірити прогноз, що вдягати під різні умови (звʼязати з правилом 3 шарів).
6. **Квіз наприкінці** — кілька запитань на закріплення (гейміфікація для підлітків). Без збору відповідей — перевірка локально.
7. **Екологія / Leave No Trace** — базові правила поведінки на природі.
8. **Типові помилки новачків / «чого НЕ брати»** — окремий короткий блок.
9. **Друк / експорт чек-листа** — щоб роздати дітям паперову версію.
10. **(Опційно) фільтр маршрутів** за днями чи складністю.

> Важливо: збір даних від користувачів (акаунти, надсилання своїх маршрутів/відповідей) НЕ планувався. Якщо така ідея зʼявиться — це вимагатиме бекенду/БД (Supabase/Firebase) і змінить архітектуру; спершу обговорити.

## Робочі нотатки / підводні камені

- Після будь-якої зміни контенту **підвищуй версію кешу** в `sw.js` (`u-gory-vN` → `u-gory-v(N+1)`) і, якщо додав нові файли (напр. фото), додай їх у масив `ASSETS`, інакше офлайн-версія буде застарілою.
- Відео з YouTube офлайн не працюють (зовнішній ресурс) — це очікувано, у застосунку є підказка.
- Локальний перегляд: подвійний клік на `index.html` працює для дизайну й чек-листа; PWA-офлайн і встановлення активуються лише через `https://` (GitHub Pages) або локальний сервер (`python3 -m http.server`).
- Деплой: GitHub Pages, Settings → Pages → Deploy from a branch, гілка `main`, тека `/ (root)`.
