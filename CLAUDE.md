# CLAUDE.md — контекст проєкту «У гори»

Цей файл — короткий бриф для Claude Code. Прочитай його на початку сесії.

## Що це за проєкт

Публічна сторінка-помічник для **підготовки до походу в гори**. Організатор походу
скидає людям лінк на сторінку — і вони бачать усе необхідне: що взяти (чек-лист),
який має бути одяг/стаф, відео та статті-поради, сервіси для маршрутів і погоди,
та головне про безпеку.

**Аудиторія:** будь-хто, хто збирається в похід (різний рівень досвіду).
Тон — простий, дружній, без зайвого тексту. Двомовно: українська + англійська.

## Ключові принципи (не порушувати без потреби)

- **Статика, без бекенду й без БД.** Жодних серверних залежностей. Дані користувача — лише в `localStorage`.
- **Mobile-first.** Основний пристрій — телефон. Великі тапи (≥44–48px), читабельність (світла тема за замовч.), є й темна тема.
- **Офлайн (PWA).** Service worker кешує все, крім зовнішніх ресурсів (відео YouTube).
- **Легке редагування контенту без коду.** Увесь контент і рядки UI — у `data.js`.
- **Хостинг — GitHub Pages.** Має працювати і при відкритті `index.html` подвійним кліком (`file://`), і на Pages.
- **Без збору даних користувачів.** Акаунти/надсилання вмісту НЕ планувались — це вимагало б бекенду; спершу обговорити.

## Технічні рішення (і чому саме так)

- **Без збірки, не монофайл.** CSS — у `css/` (кілька `.css` через `<link>` по порядку), JS — у `js/` (класичні скрипти, що ділять спільний глобальний скоуп; порядок підключення важливий, `main.js` — ОСТАННІМ). Не ES-модулі → працює `file://`. Без тулчейну.
- **Контент і мови у `data.js`.** `fetch` не працює при `file://` (CORS), тож `data.js` через `<script src>`. `window.MOUNTAIN_DATA = { uk:{...}, en:{...} }`; кожна мова має ОДНАКОВУ структуру (`ui` + контент + `safety`).
- **Стан чек-листа за ПОЗИЦІЄЮ** (`день|індекс категорії|індекс пункту`) — галочки зберігаються при зміні мови. Ключі: `gory-pack-v2`, `gory-day-v1`, `gory-season-v1`, `gory-lang-v1`, `gory-theme-v1`.
- **Експорт без бібліотек:** `js/export.js` сам генерує валідні `.xlsx`/`.docx` (store-ZIP + OOXML, кирилиця в UTF-8). PDF — через нативний друк (бо стандартні PDF-шрифти без кирилиці).
- **SW авто-оновлюється:** install → `skipWaiting`, сторінка перезавантажується на `controllerchange`. Тож після деплою користувач отримує свіжу версію на наступному візиті (без тостів/ручних дій).
- **Перевірка двомовності:** `node tools/check-i18n.js` — звіряє структуру uk/en (запускати після правок контенту).

## Структура файлів

```
mountainsApp/
├── index.html      # лише розмітка; підключає css/, data.js, js/
├── data.js         # УВЕСЬ контент + рядки UI, двомовно (uk/en)
├── css/            # base.css, components.css, safety-nav.css (порядок!)
├── js/             # state → gear → controls → render → i18n → export → main (порядок!)
├── tools/          # check-i18n.js — перевірка парності uk/en
├── sw.js           # service worker (офлайн-кеш): масив ASSETS + версія CACHE
├── manifest.json   # PWA
└── images/
    ├── icons/      # favicon + іконки PWA
    └── routes/     # фото/ілюстрації маршрутів
```

> Після додавання файлу в `css/`/`js/` — впиши його і в `index.html` (`<link>`/`<script>`), і в `ASSETS` у `sw.js`.

## Модель контенту (`data.js`, окремо в `uk` і `en`)

- **`ui`** — усі рядки інтерфейсу (таби, кнопки, заголовки, повідомлення). `brand`, `htmlLang`, `docTitle` теж тут.
- **`gear`** — `{ "1":[...], "2":[...] }` (дні). Категорія `{cat, items, must?, season?, note?}`. Пункт: `"Ложка"` або `{n, opt?, desc?}` (`opt` → бейдж «опц.», `desc` → дрібний підпис).
- **`videos`** — `{id, title, desc}` (id з YouTube).
- **`articles`** — `{title, intro?, steps:[{t,d}], warnIntro?, warnings:[{t,d}]}` (акордеон у «Поради»; steps — зелені, warnings — червоні).
- **`routes`** — `{name, area, height, duration, difficulty, days, image, desc, tip}`. **Наразі `[]`** (дані прибрано, логіка рендеру збережена). `difficulty` → колір пігулки.
- **`routeApps`** — `{name, url, tag, tagType, desc}`. `tagType`: `track`/`route`/`weather` → колір тегу (зелений/синій/помаранчевий).
- **`gearExamples`** — `{item, wrong, right, layer?}` (правильно/неправильно).
- **`safety`** — `{lead, emergency:{title, numbers:[{n,who,alt?}], notes:[]}, cards:[{title, items:[]}], footnote?}` (рендериться з даних).
- У текстах `safety`/`articles`: `**жирний**` через подвійні зірочки (рендер через `fmt()`).

## Що вже реалізовано

П'ять розділів (нижня навігація):
1. **Рюкзак** — чек-лист на 1/2 дні. Перемикач днів, перемикач «Зимове спорядження», прогрес «зібрано X/Y», авто-святкування при 100% (конфеті). Панель «Фільтр»: Усі / Необхідне / Опційне, «Сховати зібране», лічильник прихованого. Категорії авто-згортаються, коли зібрані. «Скинути» + «Експорт» (xlsx/docx/pdf). Стан зберігається.
2. **Стаф** — правило трьох шарів + 8 карток «правильно/неправильно».
3. **Поради** — відео з YouTube + статті (акордеон; перша — «Що робити, якщо вкусила змія»).
4. **Сервіси** — додатки/сервіси (трекінг/маршрути/погода) з кольоровими тегами. Маршрути зараз порожні (логіка є).
5. **Безпека** — екстрені номери (клікабельні), дії при втраті групи, перша допомога, правила групи.

Плюс: **двомовність UA/EN** (перемикач у шапці), **темна тема** (авто за системою + 🌙/☀️), PWA-офлайн з авто-оновленням, адаптивність, фокус для клавіатури, `prefers-reduced-motion`.

## Дизайн-система

- **Палітра (світла):** `--spruce:#163A2E` (бренд/шапка), `--moss:#2E6B4F` (акції/прогрес), `--signal:#F26A1B` (важливе: «Must have», екстрені), `--stone:#EEF1ED` (фон), `--paper:#FFF` (картки), `--sky:#3C7DB0` (сезонне).
- **Темна тема:** перевизначає змінні через `[data-theme="dark"]` у `base.css` (одне місце). Усі кольори — через CSS-змінні, тож тема перемикається чисто.
- **Фішки:** топо-лінії + силует гір у шапці; «блейз» (кольорова смужка) у категорій: зелений=звичайне, помаранчевий=must-have, синій=сезонне. Сміливість в одному місці, решта спокійна.

## Бэклог — ідеї, що залишилось

Пріоритет приблизний (зверху — найкорисніше).

1. **Вага рюкзака** — опційне поле ваги в пунктах (з діапазонами) + власні речі користувача + сума «≈ X кг». (Просили — наступне.)
2. **Розділ «Їжа та вода»** — скільки води, що їсти на 1 vs 2 дні, перекуси.
3. **Маршрути** — повернути/додати реальні маршрути з фото (Парашка, Хом'як, Пікуй…), позначати 1/2 дні.
4. **Розділ «Погода»** — як читати прогноз, що вдягати (звʼязати з 3 шарами).
5. **Більше статей** — перша допомога, Leave No Trace, типові помилки новачків.
6. **Квіз наприкінці** — кілька питань, перевірка локально, без бекенду.
7. **Фото-приклади в «Стаф»** — реальні зображення правильно/неправильно.

## Версіонування

Semver-теги + GitHub Releases; `main` авто-деплоїться на Pages. Повідомлення комітів і нотатки релізів — **англійською**, UI/контент — українською. Новий реліз: коміт → `git tag -a vX.Y.Z -m "..."` → `git push --tags` → `gh release create vX.Y.Z --latest ...`.

## Робочі нотатки / підводні камені

- Після будь-якої зміни **підвищуй `CACHE`** у `sw.js` (`u-gory-vN` → `v(N+1)`); нові файли — у `ASSETS`. Інакше офлайн-версія застаріла.
- SW тепер авто-оновлюється (skipWaiting + reload на controllerchange) — НЕ повертати «тихий» режим без skipWaiting, бо це лишало користувачів на старому кеші.
- Після правок контенту — `node tools/check-i18n.js` (парність uk/en).
- Відео YouTube офлайн не працюють (зовнішній ресурс) — очікувано.
- `file://` працює для дизайну/чек-листа; PWA-офлайн і встановлення — лише через `https://` (Pages) або локальний сервер.
- Деплой: GitHub Pages, гілка `main`, тека `/ (root)`.
